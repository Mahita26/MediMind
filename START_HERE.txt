━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

python train_model.py \
    --dataset torchxrayvision    # Which dataset to use
    --epochs 30                   # Training cycles (30 = good quality)
    --batch-size 32               # Images per batch (GPU: 32-64, CPU: 8-16)
    --lr 1e-4                     # Learning rate (smaller = slower but precise)
    --device cuda                 # GPU or CPU
    --patience 5                  # Stop if no improvement after 5 epochs
    --output backend/medimind_best_model.pth  # Where to save

Recommendations:
• First time: Use default settings
• GPU available: --batch-size 64 --device cuda (faster)
• Limited VRAM: --batch-size 8 --device cuda (slower but works)
• No GPU: --device cpu (much slower, like 30x slower)

Solution:

Problem: "CUDA out of memory"
Solution:
  python train_model.py --batch-size 8 --device cuda
  (Reduce batch size)

Problem: "Email validation failing"
Solution:
  • Use real email: user@gmail.com, user@company.com
  • Not: test@test.com, admin@fake.com
  • Password: 8+ chars, uppercase, digit
  • Not: "password", "12345"

Problem: "Gmail OAuth not working"
Solution:
  1. Get Google Cloud credentials
  2. Set env vars: GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET
  3. Check Google Console for redirect URI settings

Problem: "Training too slow"
Solution:
  • Use GPU: --device cuda
  • Increase batch size: --batch-size 64
  • Reduce epochs: --epochs 20 (less training = faster but less accurate)
